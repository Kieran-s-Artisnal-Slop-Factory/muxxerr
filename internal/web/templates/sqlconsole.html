Live SQL console

This runs statements against a real database — the file an app reads and writes. It is not the snapshot viewer, which is a copy in your browser and cannot affect anything.

Read this before you continue

  • There is no undo. A mistyped DELETE or DROP takes effect immediately and permanently.
  • There is no automatic backup. Nothing here snapshots the database first. If you want one, take it from your apps → Download data, before you run anything.
  • The app will be stopped. Its process is shut down before your statements run, so there is only one writer. It starts again on the next request.
  • You can break the app, not just the data. These databases have a schema the app expects. Changing it can stop the app loading at all.
